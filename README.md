# TikTok Claims Classification: Project Scoping & Proposal

## 📌 Overview
This repository contains the foundational planning and project proposal phase for the TikTok Claims Classification project. It was completed as part of Course 1 (Foundations of Data Science) in the Google Advanced Data Analytics Professional Certificate. 

The primary objective of this phase is to establish a rigorous framework for the project by defining objectives, mapping stakeholders, and establishing milestones using the **PACE (Plan, Analyze, Construct, Execute)** methodology.

## 🏢 Business Problem
TikTok users frequently report videos and comments that contain user claims, flagging them for review. The sheer volume of these reports creates a massive backlog for human moderators. 

**The Goal:** Develop a predictive machine learning model capable of accurately classifying a video as either a "claim" or an "opinion." A successful model will allow TikTok to triage and prioritize user reports efficiently, significantly reducing the moderation backlog.

## 🔄 The PACE Workflow Application
This repository focuses specifically on the **Plan** stage of the data lifecycle. The project roadmap is structured as follows:

1. **Plan (Current Phase):** Define the business problem, identify stakeholders, establish communication channels, and scope the deliverables.
2. **Analyze (Upcoming):** Perform Exploratory Data Analysis (EDA) on TikTok user engagement data, handle missing values, and select appropriate visualizations.
3. **Construct (Upcoming):** Build and validate machine learning models (e.g., Random Forest, XGBoost) to classify video transcripts and engagement metrics.
4. **Execute (Upcoming):** Evaluate model performance (Precision, Recall, F1-Score), interpret results, and present actionable insights to TikTok leadership.

## 🎯 Project Objectives & Milestones
* **Scope Definition:** Outline the exact deliverables required for the predictive model.
* **Stakeholder Alignment:** Map internal data teams and primary stakeholders (TikTok Moderation Team, Product Managers) to ensure business requirements are met.
* **Timeline Generation:** Establish clear project milestones leading up to model deployment.
